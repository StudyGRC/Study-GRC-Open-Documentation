### 1. Purpose
The purpose of this Standard is to establish a mandatory, uniform naming convention and identification system for all official governance, operational, and compliance documents within Study GRC Inc.

This standard ensures:
1.  **Retrievability:** Immediate location of documents during audits (IRS, Federal Grant, or Board review).
2.  **Version Control:** Prevention of errors caused by the use of obsolete policies.
3.  **Compliance:** Adherence to **Bylaws Article VIII (Records, Transparency & Finance)** and **Texas Business Organizations Code (TBOC) § 22.353**, which mandate the maintenance of correct and complete books and records.

### 2. Scope
This Standard applies to all "Controlled Documents" created, reviewed, or maintained by Study GRC Inc. personnel, including Board Directors, Officers, employees, volunteers, and contractors.

**In Scope:**
*   Policies, Standards, Procedures, and Guidelines.
*   Forms, Templates, and Registers.
*   Charters, Plans, and Official Reports.

**Exclusions:**
*   Temporary working drafts not yet submitted for approval.
*   Personal notes, emails, or informal correspondence.
*   External documents (e.g., invoices received from vendors) unless adopted as a template.

### 3. Definitions

| Term | Definition |
| :--- | :--- |
| **Controlled Document** | Any official document that directs behavior, processes, or governance, requiring formal approval and version control. |
| **Document ID** | A unique alphanumeric identifier assigned to a specific document to track it throughout its lifecycle. |
| **MDR** | **Master Documentation Registry**. The central database/spreadsheet where all Controlled Documents are indexed. |
| **Department Code** | A 2-3 letter prefix indicating the functional area responsible for the document. |
| **Type Code** | A 3-letter abbreviation indicating the hierarchy and enforceability of the document. |

### 4. Standards

#### 4.1 Unique Document ID Structure
All Controlled Documents must be assigned a unique Document ID following this syntax:

`[DEPT]-[TYPE]-[###]`

*   **[DEPT]:** The Department Code (see Section 4.2).
*   **[TYPE]:** The Document Type Code (see Section 4.3).
*   **[###]:** A three-digit sequential number (e.g., 001, 002, 010).

**Example:** `IS-POL-005` (Information Security - Policy - Sequence #5).

#### 4.2 Authorized Department Codes
The following codes must be used to designate the owning functional area:

| Code | Department / Domain |
| :--- | :--- |
| **GOV** | Governance & Board Operations |
| **HR** | Human Resources & Volunteer Management |
| **IS** | Information Security & IT |
| **FIN** | Finance, Accounting & Grants |
| **OPS** | General Operations, EHS & Facilities |
| **LEG** | Legal, Compliance & Intellectual Property |
| **PGM** | Programs, Education & Chapter Management |
| **COM** | Communications & Marketing |

#### 4.3 Authorized Document Type Codes
The following codes must be used to designate the document classification:

| Code | Type | Definition |
| :--- | :--- | :--- |
| **POL** | Policy | High-level rules; mandatory; "Why" and "What". |
| **STD** | Standard | Quantifiable requirements/specs; mandatory. |
| **PRC** | Procedure | Step-by-step instructions; mandatory; "How". |
| **GUD** | Guideline | Best practices; discretionary/recommended. |
| **FRM** | Form | Data collection tools (blank). |
| **TMP** | Template | Standardized layouts for content creation. |
| **PLN** | Plan | Strategic or operational roadmaps (e.g., Incident Response Plan). |
| **CHT** | Charter | Defines authority for a Committee or Department. |
| **REG** | Register | Logs or lists (e.g., Risk Register). |

#### 4.4 File Naming Convention (Digital Files)
When saving digital files (PDF, DOCX) to the central repository, the filename must adhere to the following format to ensure searchability:

`[Document ID]_[Short_Title]_v[Version].ext`

*   **Spaces:** Do not use spaces. Use underscores (`_`) or CamelCase.
*   **Short Title:** A truncated, descriptive version of the full title (max 25 characters recommended).
*   **Version:** Must match the version inside the document control block.

**Examples:**
*   `HR-POL-001_AntiHarassment_v1.0.pdf`
*   `FIN-PRC-015_CreditCardRecon_v2.1.docx`

#### 4.5 Versioning Standard
*   **Drafts:** Must use `v0.x` (e.g., v0.1, v0.2) until formally approved.
*   **Initial Release:** The first approved version is always `v1.0`.
*   **Minor Revisions:** Changes to formatting, typos, or hyperlinks that do not alter the intent or rules increment the decimal (e.g., `v1.1`, `v1.2`).
*   **Major Revisions:** Changes to scope, rules, responsibilities, or thresholds increment the whole number (e.g., `v2.0`).

#### 4.6 Document Title Requirements
*   Titles must be descriptive and written in **Title Case**.
*   Titles must not contain abbreviations unless defined in the Master Glossary.
*   Titles must avoid "legalese" (e.g., use "Employee Handbook" not "Memorandum of Understanding Regarding Employment").

### 5. Roles and Responsibilities

| Role | Responsibility |
| :--- | :--- |
| **Document Owner** | Drafts documents using the correct template; requests a Document ID from the Secretary/Compliance Officer; ensures file naming accuracy. |
| **Secretary / Compliance Officer** | Maintains the **Master Documentation Registry (MDR)**; issues the next sequential number for Document IDs; enforces this Standard during the review process. |
| **Approving Authority** | Rejects documents that do not comply with this naming standard prior to signature. |

### 6. Compliance and Enforcement
Documents submitted for approval that do not strictly adhere to this Naming Convention Standard will be **rejected** by the Approving Authority and returned to the Document Owner for correction.

Failure to maintain accurate records violates **Bylaws Article VIII** and may jeopardize the Organization’s federal grant eligibility under **2 CFR 200**. Repeated failure to comply with document control standards may result in disciplinary action.
